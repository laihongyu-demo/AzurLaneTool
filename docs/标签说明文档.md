# 数据展示标签说明文档

## 文档概述

本文档详细说明了碧蓝航线数据管理工具中所有数据展示标签的命名规范、含义解释、数据来源及展示位置，确保所有用户能够明确理解各标签所代表的具体含义。

## 标签命名规范

### 基本原则

1. **一致性原则**：同类数据使用统一的命名格式
2. **简洁性原则**：标签名称简洁明了，避免冗余
3. **可读性原则**：使用用户易于理解的术语
4. **专业性原则**：符合游戏内官方术语

### 命名格式

- 统计卡片标签：使用"主语+属性"格式，如"指挥官等级"、"舰娘总数"
- 信息标签：使用"属性名称+冒号"格式，如"图鉴ID:"、"阵营:"
- 百分比标签：使用"属性+率"格式，如"收藏率"、"誓约率"

| 标签名称 | 标签含义 | 数据来源 | 展示位置 | 数据类型 |

| 未解锁 | 未解锁的舰娘数量（含联动） | 数据库统计 | 统计面板-舰娘收藏看板 | 整数 |

## 标签展示规范

### 展示位置统一规范

1. **统计卡片**：所有统计类标签统一使用 `StatCard` 组件展示
2. **信息标签**：所有舰娘信息标签统一使用 `QFormLayout` 布局展示
3. **分组展示**：相关标签按功能分组，使用 `QGroupBox` 包裹

### 样式规范

1. **统计卡片标题**：字体大小 12px，颜色 #666
2. **统计卡片数值**：字体大小 18px，粗体，颜色 #333
3. **信息标签标题**：统一使用"标签名:"格式
4. **信息标签值**：默认显示"-"，有数据时显示实际值

## 数据来源详细说明

### 数据库表结构

#### codex_group 表
- `codex_id`: 图鉴ID（主键）
- `ship_name`: 舰娘名称
- `ship_typ`: 舰船类型
- `ship_rarity`: 稀有度
- `ship_camp`: 阵营
- `ship_level`: 觉醒等级
- `is_unlocked`: 是否解锁
- `is_oathed`: 是否誓约

#### user 配置文件 (user.json)
- `commander_level`: 指挥官等级
- `start_date`: 开始游戏日期
- `cube`: 魔方数量
- `red_gem`: 红尖尖数量

### 业务逻辑计算说明

1. **执勤天数**：当前日期 - 开始日期
2. **收藏率**：(已解锁舰娘数 / 总舰娘数) × 100%
3. **誓约率**：(已誓约舰娘数 / 已解锁舰娘数) × 100%
4. **科技点完成率**：(当前科技点 / 科技点总值) × 100%
5. **布里需求**：根据未满破舰娘的稀有度和当前突破状态计算
6. **心智单元需求**：根据可觉醒舰娘的当前等级和目标等级计算
